Test zones

One or more zones on a shared lattice, your dials. Start Hz / Separation set the lattice; each zone is Banks/zone wide, and zones are laid end-to-end with Buffer lanes of empty spectrum between them (a pollution guard). The stream visualizer spans every active zone. Applying new settings rebuilds the zones (and clears their noise profiles). Fire, Detect This, and Detect Calibration operate on zone 1.

The stream

A rolling 10-second window of the zone's full band — newest at the bottom, history rising. White is loud, black is silence; the grayscale IS what's logged (color is display-only). Faint vertical lines mark the hit-point centers; horizontal lines are the polling grid — solid when anchored to our own firing, dashed while free-running.

Start System Calibration — pattern syntax
One line = one step, fired on the polling grid on every active zone at once:
tone:MASK — an 8-bit bank mask (b1→b8 as drawn, 1 = fire) held sustained for one whole poll.
pulse:MASK — the mask fired as a short pulse (the Pulse ms set up top).
noisecancel:MS — run noise cancellation for MS milliseconds (e.g. noisecancel:1000).
blank line — one poll of silence.
All zones are scaled by Cal level scale so the simultaneous passes don't clip. Edit freely to shorten or reshape the pass.
Detect Barcode
Runs the per-zone Barcode Reader scan on the paused snapshot: Pause the stream, set the Pattern and sensitivity gates in the Barcode Reader card below, then Detect Barcode reads that mask back in each zone independently with High/Med/Low confidence. (Same action as the Reader's “Scan frozen 60 s”; results and overlays appear there.)

Barcode Reader

A tuning tool: Pause the stream, type one 8-bit bank pattern (left→right as drawn, 1=high), and scan the frozen 60 s snapshot for every duration-tall stack of it. The same mask is read in each zone independently — some zones may hit while others don't, so you can tune toward all of them. Hits are graded High/Med/Low confidence (green/amber/red); the label is the bar's best tier, the section colors show where it's strong. Raw energy only — the color/white sliders never change a result. (Separate from System Calibration; it doesn't anchor or arm.)
pause the stream, then scan
Field help
Pattern — one 8-bit mask, b1→b8 as drawn; 1 = expected loud, 0 = dark.
Reverse bank order — read the mask right-to-left (receiver/display orientation).
Bar duration — expected height of one section in ms; adjacent matching sections stack into a bar.
Allowed wrong banks — how many of the 8 bank high/low calls may disagree (per zone).
Min signal — loudness floor: a bank is "on" only if its section-average ≥ this × the zone's loudest reference.
Min shape — how well the profile matches the mask (banks high where expected, gaps cold), independent of loudness.
Max lane — ceiling on energy in the dark inter-bank gaps; catches broadband noise/bleed. Lower = stricter.
Confidence — High = your exact gates; Med/Low = progressively relaxed, so near-misses still show.

Fire

Everything fires ON the polling grid: a pulse begins exactly on the next polling line; a sequence takes one line per chord. That is deliberate — we behave the way the protocol expects, so the fingerprints land where detection will look.

Start Hz and Separation Hz are pre-filled to the live zone whenever you Enable, so sequences already fire on the detected range; change them to compare mismatched shapes against the zone's grid, or clear an override to fall back to the zone's own standards. (Chord pulse ms stays blank — it follows the zone until you set it.)

Detect This

Paste captured fingerprints into entries — one or several per entry, the thumbprint-reader model — set a confidence %, and watch colored boxes land on the stream wherever they match: live as you fire, and retroactively across the whole backlog whenever an entry changes.
